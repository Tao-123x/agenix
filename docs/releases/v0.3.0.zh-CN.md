# Agenix v0.3.0 发布说明

V0.3.0 是 Model Adapter Readiness release。

这个版本把 Agenix 从“本地 skill 能运行”推进到“adapter 选择可以在信任真实模型路径前被发现、
预检和记录”：

- 用稳定元数据列出 builtin adapters
- 在执行前预检 skill 与每个 builtin adapter 的兼容性
- 用已发布 schema 校验 adapter compatibility report
- 运行默认离线 V0.3 release gate
- 在 GitHub Actions 中覆盖 Linux、macOS、Windows
- 把 provider-backed smoke 记录为 passed、skipped 或 failed，但不放进默认 CI

## Release Gate

运行 V0.3 adapter readiness acceptance gate：

```bash
go run ./cmd/agenix acceptance --v0.3
```

预期输出：

```text
status=passed release=v0.3 adapters=5 compatibility_reports=3 schemas=3 provider_smoke=skipped_offline
```

运行显式 provider-smoke 记录：

```bash
go run ./cmd/agenix acceptance --v0.3 --provider-smoke
```

没有 `OPENAI_API_KEY` 时，预期输出是：

```text
status=passed release=v0.3 adapters=5 compatibility_reports=3 schemas=3 provider_smoke=skipped_no_credentials
```

有凭据时，该命令会运行只读 `openai-analyze` adapter，并输出
`provider_smoke=passed` 和 `provider_smoke_trace=<trace-path>`。

离线 release gate 还包括：

```bash
go run ./cmd/agenix acceptance
go run ./cmd/agenix acceptance --v0.2
go test -count=1 ./...
go vet ./...
go build ./cmd/agenix
```

## 这次变化

- 新增 `agenix adapters` 和 `agenix adapters --json`。
- 新增 `agenix adapters compatible <manifest|artifact|skill@version>`。
- 给 adapter compatibility preflight 增加 registry-reference 支持。
- 新增已发布契约 `specs/adapter-compatibility-report.schema.json`。
- 扩展 `agenix validate`，让它能识别 adapter compatibility report。
- 新增 `agenix acceptance --v0.3` 作为 adapter readiness release gate。
- 新增 `agenix acceptance --v0.3 --provider-smoke` 作为显式 provider smoke 记录路径。
- GitHub Actions 覆盖 `ubuntu-latest`、`macos-latest` 和 `windows-latest`。
- 在没有 `go.sum` 时关闭 setup-go cache，并让 workflow opt into Node 24 actions
  runtime。

## 跨 OS 证据

`v0-release-gate` workflow 会在以下系统运行同一套离线 gate：

- Linux：`ubuntu-latest`
- macOS：`macos-latest`
- Windows：`windows-latest`

release gate 是可移植性证据入口。只有本地 macOS 通过，不足以验证 V0.3。

## 边界

V0.3.0 不新增 marketplace、公共 registry、强 sandbox、签名、OCI 分发、streaming
provider output、provider-side tool calling 或默认远程执行。Provider-backed 路径仍然是
opt-in，并且只用于 canonical remote-analysis skill 的只读路径。
