# Agenix v0.3.0 Release Notes

V0.3.0 is the Model Adapter Readiness release.

The release moves Agenix from "local skills can run" to "adapter choices can be
discovered, checked, and recorded before a real model path is trusted":

- list builtin adapters with stable metadata
- preflight a skill against each builtin adapter before execution
- validate adapter compatibility reports with a published schema
- run a default offline V0.3 release gate
- run the release gate on Linux, macOS, and Windows in GitHub Actions
- record provider-backed smoke as passed, skipped, or failed without making it
  part of default CI

## Release Gate

Run the V0.3 adapter readiness acceptance gate:

```bash
go run ./cmd/agenix acceptance --v0.3
```

Expected output:

```text
status=passed release=v0.3 adapters=5 compatibility_reports=3 schemas=3 provider_smoke=skipped_offline
```

Run the explicit provider-smoke record:

```bash
go run ./cmd/agenix acceptance --v0.3 --provider-smoke
```

Without `OPENAI_API_KEY`, expected output is:

```text
status=passed release=v0.3 adapters=5 compatibility_reports=3 schemas=3 provider_smoke=skipped_no_credentials
```

With credentials, the command runs the read-only `openai-analyze` adapter and
prints `provider_smoke=passed` plus `provider_smoke_trace=<trace-path>`.

The offline release gate also includes:

```bash
go run ./cmd/agenix acceptance
go run ./cmd/agenix acceptance --v0.2
go test -count=1 ./...
go vet ./...
go build ./cmd/agenix
```

## What Changed

- Added `agenix adapters` and `agenix adapters --json`.
- Added `agenix adapters compatible <manifest|artifact|skill@version>`.
- Added registry-reference support to adapter compatibility preflight.
- Added the published
  `specs/adapter-compatibility-report.schema.json` contract.
- Extended `agenix validate` to recognize adapter compatibility reports.
- Added `agenix acceptance --v0.3` as the adapter readiness release gate.
- Added `agenix acceptance --v0.3 --provider-smoke` as the explicit provider
  smoke record path.
- Added GitHub Actions coverage for `ubuntu-latest`, `macos-latest`, and
  `windows-latest`.
- Disabled setup-go cache when `go.sum` is absent and opted the workflow into
  the Node 24 actions runtime.

## Cross-OS Evidence

The `v0-release-gate` workflow runs the same offline gates on:

- Linux: `ubuntu-latest`
- macOS: `macos-latest`
- Windows: `windows-latest`

The release gate is the portability evidence path. A local macOS run alone is
not enough to validate V0.3.

## Boundary

V0.3.0 does not add a marketplace, public registry, stronger sandbox, signing,
OCI distribution, streaming provider output, provider-side tool calling, or
default remote execution. The provider-backed path remains opt-in and read-only
for the canonical remote-analysis skill.
