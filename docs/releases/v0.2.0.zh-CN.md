# Agenix v0.2.0 发布说明

V0.2.0 是 Skill Authoring release。

这个版本把 Agenix 从“canonical demo 能跑”推进到“一个新 skill 作者有可重复路径”：

- 列出内置 authoring 模板
- 从模板初始化 skill
- 校验生成的 manifest
- 构建可移植 `.agenix` artifact
- 通过显式本地 adapter 运行 artifact
- 用 `agenix check` 作为一条命令的 authoring gate
- 校验成功和失败两种 `check_report` JSON

## Release Gate

运行 V0.2 authoring acceptance gate：

```bash
go run ./cmd/agenix acceptance --v0.2
```

预期输出：

```text
status=passed release=v0.2 templates=2 skills=2 checks=3 failure_reports=1
```

这个 gate 覆盖：

- `python-pytest` 模板生成和只读执行
- `repo-fix-test-failure` 模板生成和受限修复执行
- 生成 skill 的 manifest 校验
- 生成 skill 的 artifact build、inspect、run
- 通过的 `agenix check` 报告
- 失败的 `agenix check` 报告，包含 `error_class`、`error_message`、trace path
  和非零 CLI 行为
- 生成的 `check_report` 文档 schema 校验

## 这次变化

- 新增 `agenix init templates` 和 `agenix init templates --json`。
- 新增 `agenix init skill <name> --template <template> -o <dir>`。
- 新增 `agenix check <skill-dir|manifest|artifact> [--adapter <name>] [--json]`。
- 新增已发布契约 `specs/check-report.schema.json`。
- 扩展 `agenix validate`，让它能识别 check report。
- 新增失败 check 的 JSON report，让 CI 和其他 agent 在 gate 失败时也能拿到结构化证据。
- 新增第一篇写 skill 的新手教程。

## 边界

V0.2.0 仍然限定在本地 reference runtime 内。它不新增远程执行器、registry marketplace、
强 sandbox、签名，或默认 provider-backed 模型执行。这个版本的目标是可靠的 authoring
闭环，不是扩大信任边界。
